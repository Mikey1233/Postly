# Product

## Register

product

## Users

One person — Postly's owner. A solo content creator who posts daily across LinkedIn, Facebook, X, and Reddit. Sits at a desk, often late at night, often on two monitors. Treats Postly as a daily-driver tool, not a campaign space. LinkedIn quality matters more than the other three combined.

## Product Purpose

Personal social media manager. Write, schedule, and publish to four platforms from one interface, without context-switching. LinkedIn-first AI — voice analyzer, carousel builder, post scoring, ghost-text autocomplete — tuned for serious, on-brand posts. Success is fewer apps open, more posts shipped, every one in the owner's voice.

## Brand Personality

Precise. Resourceful. Decisive. Workshop, not playground. Daily-driver feel — the owner lives inside it, so it must reward repeat use with density, hotkeys, and zero ceremony. Quiet by default; loud only at the moments that matter.

## Anti-references

- ChatGPT-style chat UI — message bubbles, alternating L/R columns, avatar circles, gray right-side info panel, "AI says…" framing.
- SaaS dashboard reflexes — card grids of identical tiles, hero-metric headers, sidebar + topbar + main everywhere, "Welcome back!" banners.
- AI-startup neon — gradient text, purple/pink glow, glassmorphism as default, sparkles on every button, decorative motion.
- Light-mode-by-default productivity tools that ignore the user's actual environment.

## Design Principles

1. **Tools over chrome.** Every pixel earns its place. Chrome compresses; content expands. If a label restates the obvious or a card wraps a single sentence, cut it.
2. **Power-user defaults.** Keyboard-first. Density without clutter. Hotkeys visible inline so they're learnable, not hidden in a help dialog.
3. **The artifact is the reward.** The workspace stays quiet so the reveal moments — a finished carousel, a scored post, a successful publish — can be loud. Don't spend the visual budget early.
4. **One committed color.** Indigo is Postly's signature, carrying 30–60% of the interactive chrome (active states, primary buttons, focus rings, accent rules). Everything else is a near-black neutral tinted toward that same hue. No second accent without reason.
5. **No chat-UI clichés.** Conversations with AI are workshop logs, not chat threads. Same column, same rhythm. The user is not "messaging the agent" — they are directing a tool.

## Accessibility & Inclusion

- Target WCAG AA across all surfaces (contrast, focus indicators, keyboard reachability).
- Respect `prefers-reduced-motion` for the artifact slide-in, tray expansion, and any non-essential transitions — collapse to instant when set.
- Single-user product, so no i18n burden.
- All hotkeys must have a non-keyboard fallback (toolbar button, click target) so the UI is usable without memorizing shortcuts.
